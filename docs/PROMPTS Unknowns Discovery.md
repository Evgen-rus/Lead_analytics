# Готовые команды

## Инициализировать проект

```text
Используй $unknowns-discovery в режиме bootstrap.
Изучи репозиторий и заполни подтверждаемую часть docs/agent/PROJECT_CONTEXT.md.
Не пиши код. Не выдумывай отсутствующие сведения.
После исследования задай мне по одному не более 7 критичных вопросов.
```

## Найти слепые зоны задачи

```text
Используй $unknowns-discovery в режиме discovery.

Задача:
[вставить задачу]

Сначала изучи релевантный код и документы.
Код не меняй.
Найди неизвестные, которые могут изменить архитектуру, данные,
пользовательский сценарий, достоверность или стоимость переделки.
После отчёта задавай вопросы по одному.
```

## Сделать план

```text
Используй $unknowns-discovery в режиме plan.
Учти мои ответы и зафиксируй решения в docs/agent/decisions/.
Код не пиши.

Порядок плана:
1. решения;
2. пользовательский сценарий;
3. изменения данных и контрактов;
4. риски и совместимость;
5. критерии приёмки;
6. технические шаги и файлы.
```

## Проверить аналитический вывод

```text
Используй $unknowns-discovery в режиме evidence-audit.

Материал:
[сделка, отчёт, статусы, транскрибации или вывод]

Раздели:
- факты;
- интерпретации;
- неизвестные;
- альтернативные объяснения;
- что проверить;
- как неизвестные влияют на вердикт;
- уверенность.

Не считай CRM-статус или мнение менеджера объективным фактом без подтверждения.
```

## Проверить новый продукт

```text
Используй $unknowns-discovery в режиме product-discovery.

Идея:
[описание]

Не проектируй систему.
Найди самые рискованные предположения.
Для каждого предложи самый дешёвый тест и конкретный сигнал успеха/опровержения.
```

## Реализовать после согласования

```text
Используй $unknowns-discovery в режиме implementation.
Реализуй утверждённый план.
Веди implementation-notes.md.
Низкорисковые обратимые решения принимай самостоятельно и фиксируй.
Не меняй внешние контракты, данные и архитектуру молча.
После работы покажи отклонения, тесты, ограничения и оставшиеся неизвестные.
```

## Проверить готовую работу

```text
Используй $unknowns-discovery в режиме review.
Сравни реализацию с целью, решениями и критериями приёмки.
Найди неподтверждённые предположения, edge cases и пробелы тестирования.
Сначала покажи проблемы, затем конкретные исправления.
```
